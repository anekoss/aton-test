# Тестовое задание в команду корпоративной шины данных и микосервисов

Для реализации In memory DB использована структура данных `Map`. Для обеспечения быстроты операций `-> O(1)` выбрана реализация `HashMap`.

Поле account считается уникальным для каждого человека.

```java
Map<Long, Person> personAccounts = new HashMap<>();
```

**Поиск:** O(1)
Для быстрого поиска использованы карты с ключем-полем.

Рекоммендуемая реализация List - ArrayList для оптимизации памяти.
```java
Map<String, List<Person>> personNames = new HashMap<>();
Map<Double, List<Person>> personValues = new HashMap<>();
```
**Добавление:** O(1): в HashMap за O(1), в List за O(1) в конец  

**Удаление:** O(n): в HashMap за O(1), в List за O(n), где n - длина листа

**Обновление:** O(n), где n - длина листа

При обновлении сначала происходит удаление, затем вставка.

**Память:** O(n)

В java все объекты хранятся по ссылке, т.е. в HashMap и List будут храниться ссылки на созданный объект.

**Тестирование:**

Для тестирования созданы HashMap, заполненные тестовыми данными. Написаны тесты на все методы класса InMemoryDb.
